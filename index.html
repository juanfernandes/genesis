The Grid

The base grid has 16 columns and is based on a 960 grid system. Unlike most grid systems, each column based on percentages rather than pixel dimensions. The syntax is simple and it's effective cross browser, but the best part is that it has a lot of flexibility to fit any viewport size, current or future.

Supported form controls

Form controls can help build support to any layout.

Inputs

Most common form control are the text-based input fields. Includes support for all HTML5 types: text, password, datetime, datetime-local, date, month, time, week, number, email, url, search, tel, and color. Please note that not all browsers support of these input types. Check the browser adoption before use.

Requires the use of a specified type= at all times.

Textarea

A standard form control that will support multiple lines of text. Change the number of rows to fit your needs.

Form actions

Action elements are essential to help users finalize any form. When placed within a .form-horizontal, the buttons will indent to line up with the form controls.

Form control states

Displaying how to provide feedback to users with basic feedback states on form controls and labels can be quite helpful and should be thought about before your final build.

Input focus

Removing the default .outline style on some form controls and apply the .box-shadow in its place will offer better control over the :focus.

Disabled inputs

Add the .disabled attribute on an input to prevent user input.

Validation states

Showing validation states can help better communicate situations that users will have to deal with like error, warning, info, and success messages. To use, add the appropriate class to the surrounding .control-group.
